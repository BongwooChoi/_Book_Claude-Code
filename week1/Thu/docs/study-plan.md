## 진행 시 주의사항

1. **트러블슈팅이 최대 리스크.** 2021년 출간 도서라 저장소 주소·이미지 정책이 여럿 바뀜. 에러 발생 시 30분 이상 혼자 붙잡지 말고 README 알림 → docs 트러블슈팅 PDF → 공식 저장소 이슈 순으로 확인. 매주 일요일이 버퍼인 이유.
2. **매 실습 종료 시 `kubectl get all`로 상태 확인.** 챕터가 끝나면 `vagrant destroy`로 정리 후 다음 챕터 환경을 새로 올릴 것. 이전 챕터 잔재가 다음 실습 에러의 흔한 원인.
3. **토요일 셀프 테스트(8/1, 8/15)가 핵심.** 책을 덮고 재현이 되는지가 실제 학습 여부의 기준.
4. RAM 16GB 미만이면 ch3 클러스터 배포가 느리거나 실패할 수 있음. 이 경우 Vagrantfile에서 워커 노드 수를 3 → 2로 줄이는 것도 방법.
